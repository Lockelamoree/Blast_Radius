# Final screenshot capture checklist

The three `screen_*.png` files are live 1200×800 captures from the deployed app. If the UI
changes materially, re-capture using the checklist below and keep the filenames so the README and
Devpost links remain valid.

1. `screen_decision.png` — show the Decision Card with an agent artifact, all three actions,
   and the evidence-tell prompt. Keep the artifact readable.
2. `screen_verdict.png` — show a graded Verdict Stamp, tell coverage, gate/provenance line, and
   at least one direct receipt. This is the primary proof image.
3. `screen_results.png` — show the results/mastery view with pre-test, post-test, delta, and the
   competency map. Do not imply a named learning result unless the tester consented.

Capture from `https://blastradius.max-gutowski.de/` at the deployed revision. Use the same
browser width for all three, hide unrelated browser chrome if possible, and confirm no private
access code, cookie, recovery token, or developer-only data is visible.
